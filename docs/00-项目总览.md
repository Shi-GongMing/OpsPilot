# OpsPilot 项目面试教程 — 总览

> **阅读指南**：本文档是 OpsPilot 项目面试准备的入口。详细模块讲解见 `01-10` 号文档。标注 `[自研]` 的概念建议自行查阅，本文不做展开。

---

## 项目一句话定位

**具备自演进能力的智能运维 Agent 系统——每次诊断完自动归档经验到向量库，系统越用越聪明。**

---

## 两大核心能力

| 能力 | 模式 | 技术路线 | 价值 |
|------|------|---------|------|
| ReAct 智能对话 | 用户提问 → Agent 自主选工具 → SSE 流式返回 | LangGraph `create_agent()` + Function Calling | 运维工程师自然语言查日志/告警/历史案例 |
| AIOps 故障诊断 | 告警触发 → Plan-Execute-Replan 多步推理 → 根因报告 | 手写 LangGraph StateGraph 三节点架构 | 从告警到根因全流程自动化，含三层安全收敛阀 |

---

## 核心创新：自演进知识闭环

```
诊断触发 → Planner 检索历史案例 → Executor 调工具收集证据
    → Replanner 评估决策 → 生成最终报告 → 自动归档到 Milvus
    → 下次 Planner 检索时命中归档案例 → 诊断越来越准
```

这不是"人工上传运维文档"——这是 **Agent 自主积累经验**。每一次诊断都在增强系统能力。

## 技术栈速览

| 层 | 技术 | 用途 |
|---|------|------|
| Web 框架 | FastAPI + Uvicorn | HTTP API + SSE 流式 |
| Agent 框架 | LangGraph (`create_agent` + 手写 StateGraph) | ReAct 循环 + Plan-Execute-Replan |
| LLM | 阿里 DashScope qwen3.7-plus (OpenAI 兼容) | 对话生成、计划制定、步骤执行、摘要压缩 |
| Embedding | DashScope text-embedding-v4, 1024 维 | 文档/摘要向量化 |
| 向量库 | Milvus (L2 索引) | 知识检索 + 归档写入 |
| 工具协议 | MCP (Model Context Protocol) | 标准化接入日志/监控外部工具 |
| 会话持久化 | LangGraph AsyncSqliteSaver + aiosqlite | 跨重启对话保留 |
| 通信协议 | SSE (Server-Sent Events) | 单向实时推送 Agent 思考过程 |
| 前端 | 原生 JS + marked.js + highlight.js | Markdown 渲染 + 代码高亮 |
| 日志 | Loguru | 结构化日志，按天轮转 |

## 面试时如何介绍项目

**30 秒版本**（一面自我介绍）：
> "我主导开发了一个叫 OpsPilot 的智能运维 Agent 系统。它有两个核心能力：一是运维工程师能用自然语言查日志查告警查解决方案，Agent 自主决策调工具；二是遇到新告警时自动走 Plan-Execute-Replan 流程诊断根因。最核心的设计是诊断完自动归档经验到向量库，系统排查的故障越多经验越厚。"

**3 分钟版本**（二面深入）：
> 1. **先说架构决策**："我在这做了几个关键的架构决策。第一个是对话用 `create_agent()` 而诊断用手写 StateGraph——因为诊断需要自定义三节点拓扑。第二个是通信协议选了 SSE 而不是 WebSocket——因为 Agent 流式是天然的单向推送场景，SSE 基于 HTTP，浏览器原生自动重连，代理零配置。"
> 2. **再说工程亮点**："安全收敛这块我们设计了三层阀——Agent 有时候会无限重规划停不下来，我做了步数硬上限、决策软约束、计划膨胀抑制三道防线。延迟方面把千问的思维链关掉了，实测首字从 10 秒降到 1 秒。"
> 3. **最后讲闭环**："每次诊断完自动把故障报告归档到 Milvus，下次遇到类似问题 Planner 直接检索到历史经验。我们做了 L2 距离阈值过滤加时间衰减重排序，把噪声过滤掉，把近期的案例排前面。"
