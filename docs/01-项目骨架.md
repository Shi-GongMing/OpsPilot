# 01 — 项目骨架：如何启动一个 Agent 系统

> **面试重点**：FastAPI lifespan 管理资源生命周期、Pydantic Settings 配置管理
>
> **涉及文件**：`main.py`、`config.py`、`utils/logger.py`、`app/__init__.py`

---

## 模块加载链路

```
uvicorn app.main:app
  → app/__init__.py          配置日志（最早执行）
  → app/main.py              导入所有模块
    → config.py              解析 .env → Settings 单例
    → api/chat.py            导入 RagAgentService
    → services/rag_agent_service.py  创建 Agent 单例
    → ...
  → lifespan 启动            连接 Milvus + AsyncSqliteSaver
  → 应用就绪，监听 :9900
```

这条链路上的关键设计：**日志最先初始化**（`__init__.py` 先于所有业务模块执行），**配置在导入时完成**（`config = Settings()` 是模块级语句），**外部资源在 lifespan 中连接**（启动时连，关闭时断）。

## 1. 配置管理 — Pydantic Settings

### 为什么不用 os.getenv() 散落各处

`os.getenv()` 的问题是：类型不安全（都是字符串）、拼写错误在运行时才暴露、IDE 无法自动补全、修改需要全局搜索。

Pydantic Settings 的方案：定义一个 `Settings` 类，字段带类型注解。`.env` 文件中的字符串自动转换为对应 Python 类型：

```
.env                         →   config 对象
DASHSCOPE_API_KEY=sk-xxx     →   config.dashscope_api_key = "sk-xxx"  (str)
MILVUS_PORT=19530            →   config.milvus_port = 19530            (int)
DEBUG=True                   →   config.debug = True                  (bool)
```

### 关键配置

```python
model_config = SettingsConfigDict(
    env_file=".env",          # 读取文件
    case_sensitive=False,     # DASHSCOPE_API_KEY = dashscope_api_key
    extra="ignore",           # 未定义字段不报错
)
```

`mcp_servers` 属性将分散的扁平配置聚合为 MCP 客户端需要的嵌套字典——`.env` 保持简单可编辑，代码中按需组装。

### [自研概念] Pydantic Settings / SettingsConfigDict

## 2. 应用生命周期 — `@asynccontextmanager`

```python
@asynccontextmanager
async def lifespan(app):
    milvus_manager.connect()         # 启动
    yield                            # ← 应用运行中
    milvus_manager.close()           # 关闭
    await close_checkpointer()
```

**为什么用 lifespan 而不是 `@app.on_event("startup")`**：lifespan 是 FastAPI 推荐的现代化写法，所有启动逻辑在一个代码块内，启动和关闭对称可见。`yield` 前是启动阶段，`yield` 后是关闭阶段。如果启动失败（如 Milvus 不可达），异常在 `yield` 前抛出，应用不会进入就绪状态。

### 面试一句话

> "我在 lifespan 里管理外部资源——Milvus 连接和 SQLite 连接都在启动时建立，关闭时释放。这样保证了请求到达之前依赖已就绪，应用退出时不会有连接泄漏。"

## 3. 日志 — Loguru

**为什么不选 Python 标准 logging**：Loguru 开箱即用——自带颜色、按天轮转、异常自动追踪变量值。logging 需要手动配置 Handler/Formatter/Filter 四件套。

配置要点：
- `enqueue=True`：多进程安全，日志写入由独立线程排队
- `rotation="00:00"`：每天午夜切新文件
- 在 `app/__init__.py` 中初始化——任何模块导入前日志已经就绪

## 4. 路由注册与静态文件

每个 API 模块独立文件，通过 `include_router` 挂载：

```python
app.include_router(chat.router, prefix="/api")   # /api/chat, /api/chat_stream
app.include_router(aiops.router, prefix="/api")  # /api/aiops
```

`prefix="/api"` 让模块内部写 `/chat` 即可，外部路径自动变为 `/api/chat`。

静态文件的挂载用 `mount`（不是 `include_router`）——mount 是纯文件服务，不走路由逻辑：
```python
app.mount("/static", StaticFiles(directory="static"))
```

## 5. CORS

开发环境放通所有来源 `allow_origins=["*"]`。生产环境应限制具体域名。这不是项目重点，但面试时如果能提一句"生产环境不会用 `*`"，说明你有安全意识。
