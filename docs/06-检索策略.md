# 06 — 检索策略：如何"搜得准"

> **面试重点**：L2 距离 vs 余弦相似度的工程选择、阈值过滤 + 时间衰减的混合排序
>
> **涉及文件**：`tools/knowledge_tool.py`、`services/vector_index_service.py`

---

## 1. L2 距离 vs 余弦相似度：先搞清楚度量

Milvus 的默认索引使用的度量是 **L2 距离**（欧几里得距离），不是余弦相似度。这很重要——两者的数值含义完全不同：

| 度量 | 范围 | 越好越 | 好结果 | 噪声 |
|------|------|--------|--------|------|
| L2 距离 | [0, +∞) | 低 | <1.0 | >1.0 |
| 余弦相似度 | [-1, 1] | 高 | >0.75 | <0.5 |

**如果你跟面试官说"我们设了余弦相似度阈值 0.75"，但 Milvus 的索引是 L2——他立刻知道你没看过自己数据库的索引配置。** 这是基础问题，必须匹配。

我们的实测数据（text-embedding-v4, 1024 维）：

```
"CPU使用率过高" → cpu 文档:      L2=0.62 ✓ 强匹配
"内存溢出排查"  → memory 文档:    L2=0.63 ✓ 强匹配
"磁盘空间不足"  → disk 文档:      L2=0.77 ✓ 强匹配
"你好世界"      → 运维知识库:     L2=1.39 ✗ 完全无关
```

结论：**L2 < 1.0 是有意义的结果，L2 > 1.0 是噪声**。阈值定在 1.0。

### 面试一句话

> "阈值不是拍脑袋定的——我们实测了 text-embedding-v4 在这个场景下的距离分布。强匹配结果 L2 在 0.5-0.8，弱相关 0.8-1.0，噪声全部在 1.0 以上。所以我们把阈值设在 1.0。"

## 2. 检索全链路

```
query → Embedding → Milvus L2 搜索 (fetch_k = top_k × 3)
  → L2 距离过滤 (L2 < 1.0，剔除噪声)
  → 相似度转换: similarity = 1 / (1 + L2)
  → 时间衰减加权: × e^(-0.023 × days)
  → 组合排序 → 返回 TopK
```

---

## 3. 为什么拉取 3 倍候选

`fetch_k = top_k × 3`。因为过滤 + 重排后实际返回数量不确定——如果 Top-3 全被过滤掉（噪声查询），最终返回 0 条（而不是强行塞 3 条低质量结果给 LLM）。多拉候选保证了过滤后仍有足够高质量的条目。

---

## 4. 时间衰减加权

公式：`final_score = similarity × e^(-0.023 × days)`

| 文档年龄 | 权重 |
|----------|------|
| 今天 | 1.00 |
| 7 天 | 0.85 |
| 30 天 | 0.50（减半） |
| 90 天 | 0.13 |

**衰减系数 0.023 的含义**：`ln(2)/30 ≈ 0.023` → 30 天权重降为一半。这个值不是随便选的——它是"运维知识的半衰期"估计。如果你认为运维经验的有效期更长或更短，调整 λ 即可。

### 面试一句话

> "单纯的相似度检索有个问题——三年前的故障案例可能和当前问题语义匹配度很高，但解决方案可能已经过时了。我们在相似度基础上乘以指数时间衰减因子，让近期的案例自动排前面。衰减系数 0.023 意味着 30 天权重减半。"

---

## 5. L2 → 相似度转换公式

`similarity = 1 / (1 + L2)`

| L2 | → | 相似度 |
|----|---|--------|
| 0.5 | → | 0.67 |
| 0.8 | → | 0.56 |
| 1.0 | → | 0.50 |
| 1.5 | → | 0.40（已被过滤） |

---

## 6. 噪声查询验证

```
Query: "今天天气真好"（与运维完全无关）
  → 候选 9 条 → L2 过滤 9 条 → 返回 0 条
  → Agent 收到 "没有找到相关信息"
```

对比旧版（无过滤）：同样查询返回 3 条低质量文档，LLM 基于这些噪声内容生成不准确的回答。**这就是"减少 LLM 幻觉"的具体工程手段。**

---

## 面试话术

> "我们在检索上做了两个优化。第一是 L2 距离阈值过滤——实测我们用的 text-embedding-v4 在运维场景下，强匹配 L2 < 0.8，噪声 L2 > 1.0，所以阈值设在 1.0。低于阈值的直接丢弃，不给 LLM 看到，从源头减少幻觉。第二是时间衰减加权——运维经验有时效性，上个月的案例比去年的更有参考价值。组合打分 = L2 相似度 × e^(-λ·t)，30 天权重减半。"
